# 🤝 Contributing to Uphonic

Thank you for your interest in contributing to **Uphonic**! Whether you're fixing bugs, adding features, or improving documentation, your help is appreciated.

---

## 🧰 Getting Started

1. **Fork the repository** and clone it to your local machine.
2. Ensure you can build the project:
   ```bash
   generate.bat
   build.bat
   ```

3. Create a new branch for your changes:
    ```bash
    git checkout -b feature/my-feature
   ```
4. Make your changes and commit with clear messages:
    ```bash
    git commit -m "Add feature: XYZ"
    ```
5. Push your branch and open a pull request.

## 📋 What You Can Work On
- Bug fixes
- Features listed in our [**README**](README.md)
- Refactoring and code cleanup
- Documentation updates
## 🧼 Code Style
- Stick to **C-like C++** conventions
- Keep modules minimal and scalable
- Avoid introducing heavy dependencies
- Any third-party library must be licensed under GPLv3 or a more permissive license (e.g., MIT, BSD).
> We do not accept libraries with stricter or less compatible licensing terms than GPLv3.

## 💬 Communication
- Open an issue for discussion before large changes
- Stay respectful and constructive
- Keep PRs focused and clear
